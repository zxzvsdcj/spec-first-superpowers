版本：2.0
批准日期：YYYY-MM-DD
适用范围：本项目所有规范、计划、任务、代码均必须遵守此宪法。
违反任何条款均为 blocker，需要立即修正。

**本宪法在每个质量闸门(G1-G4)中被主动校验。plan.md 必须引用检查点。**

## 1. 前言与目的

本宪法定义项目的**不可妥协原则**，是 SDD 流程的最高指导文件。
AI 在生成 spec.md / plan.md / tasks.md 时必须引用并遵守这些原则。

## 2. 核心原则

### 2.1 代码质量 <!-- G4 校验 -->

- 可读、可维护、可测试（Clean Code）
- 公共 API 完整文档
- SOLID · DRY · KISS
- 禁止魔法数字/字符串
- 命名规范统一
- 代码风格工具强制（pre-commit hook）
- 禁止循环依赖，清晰分层
- 所有变更通过代码审查

### 2.2 测试规范 <!-- G4 校验 -->

- TDD / BDD 优先
- 覆盖率：关键路径 ≥ 95%，整体 ≥ 85%
- 测试独立、可重复、无副作用
- CI/CD 测试失败 → 阻塞合并
- 禁止提交未测试代码

### 2.3 用户体验一致性 <!-- G3 校验 -->

- 统一设计系统（Atomic Design）
- 响应式 (mobile-first)
- WCAG 2.1 AA（ARIA · 键盘导航 · 对比度 ≥ 4.5:1）
- 交互一致（按钮/导航/加载/错误统一）
- 国际化支持
- 页面加载 < 3s，交互 < 100ms

### 2.4 性能要求 <!-- G4 校验 -->

- 首屏 < 2s (P90)
- 主包 < 500KB (gzip)
- 避免 N+1 查询
- API < 200ms (P95 < 500ms)
- 监控 LCP/FID/CLS

## 3. 架构治理 <!-- G1 校验 -->

- 重大架构决策记录在 plan.md，**引用本宪法条款**
- 优先选择成熟、社区活跃、LTS 技术
- 禁止高风险依赖
- 第三方库许可证审查（MIT/Apache 优先）
- OWASP Top 10 防护

## 4. 文档分离 <!-- G1 校验 -->

- spec.md：**纯产品视角** (What & Why)，禁止框架/库/代码结构
- plan.md：**工程视角** (How)，必须引用 constitution 检查点
- 违反分离 → blocker

## 5. 闸门检查点速查

| 闸门 | 校验条款 |
|------|---------|
| G1 | §3(架构治理) §4(文档分离) |
| G2 | plan.md 是否引用 constitution 检查点 |
| G3 | §2.3(用户体验一致性) |
| G4 | §2.1(代码质量) §2.2(测试) §2.4(性能) |

## 6. 变更与修订

- 修订需记录版本历史 + 理由
- 修订后 AI 自检现有 spec/plan 是否仍符合
- 建议每季度或重大技术变更时审视
