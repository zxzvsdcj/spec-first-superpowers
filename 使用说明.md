# 使用说明

## 1. 系统目标

强制 AI 编程助手先完成规范（SDD）再写代码，杜绝"凭感觉乱写"。
集成五个核心工具：Spec-Kit + OpenSpec + Superpowers + planning-with-files + ui-ux-pro-max。

**v3 核心升级**：复杂度分级 · 会话恢复 · 质量闸门 G0-G4 · 两阶段审查 · 错误升级链 · 5 条协同链。

## 2. 安装

### 外部 CLI

```bash
chmod +x install-all.sh && ./install-all.sh
```

### Cursor Skills

在 Cursor 中确保已安装：
- `using-superpowers`（含子 Skills）
- `planning-with-files`
- `finishing-a-development-branch`
- `ui-ux-pro-max`（推荐）

### 文件放置

| 文件 | 放置位置 |
|------|----------|
| `SKILL.md` + `references/` + `assets/` | `~/.cursor/skills/spec-first-superpowers/` |
| `00-spec-first-superpowers.mdc` | `<project>/.cursor/rules/` |
| 宪法模板（可选） | `.specify/memory/` 或 `.openspec/` |

## 3. 使用方式

### 启动

```
/super-spec
```

或带参数：

```
/super-spec force-openspec 添加用户仪表盘
```

### 完整流程

1. **复杂度分级** — AI 自动判断快速/标准/深度，用户确认
2. **上下文恢复** — 检测已有 task_plan.md 时自动恢复
3. **SDD 规范** — 按模式执行规范流程，闸门 G1（用户确认+宪法对齐）
4. **持久规划** — 生成 task_plan.md 等三文件，闸门 G2（编号清单+测试点）
5. **UI/UX 设计** — 条件触发，`--persist` 持久化，闸门 G3
6. **TDD 执行** — Subagent-Driven 或 Executing-Plans，闸门 G4（测试+审查+证据）
7. **归档** — finishing-a-development-branch + 更新全部文件

### 典型示例

```
你：/super-spec 新增登录页面，支持邮箱+密码和 Google 登录
AI：[复杂度=标准] → OpenSpec 规范 → 持久规划 → UI/UX 设计系统 → TDD 实现
```

## 4. 常见问题

| 问题 | 解决 |
|------|------|
| 未触发？ | 检查 rules/skills 路径，重启 Cursor |
| 模式错误？ | `/super-spec reset` 重置 |
| UI 未触发？ | 需求中明确写界面关键词 |
| 上下文丢失？ | 检查 task_plan.md / progress.md 是否最新 |
| 3 次错误？ | 自动调用 systematic-debugging |
