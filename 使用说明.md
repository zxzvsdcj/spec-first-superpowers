### AI 编程终极约束系统完整使用说明（Spec-First + Superpowers 终极版）

#### 1. 系统目标与核心价值
构建一套**强制约束 AI 编程助手**的工作流，确保：
- 先完成规范（SDD）再写代码，杜绝“凭感觉乱写”
- 默认偏向 **OpenSpec**（轻量、迭代友好，适合大多数存量项目）
- 集成文件持久规划（planning-with-files）
- 自动生成专业 UI/UX 设计系统（ui-ux-pro-max-skill）
- 强制 TDD + 代码审查 + 验证（Superpowers 方法论）
- 支持一键激活命令：`/super-spec`

#### 4. 安装步骤（一键脚本）
在项目根目录创建并运行 `install-all.sh`（完整脚本见会话历史）：

```bash
chmod +x install-all.sh && ./install-all.sh
```

脚本会自动安装上述所有工具，并创建：
- 自定义 meta-skill（SKILL.md）
- Cursor 规则文件（.cursor/rules/00-spec-first-superpowers.mdc）

#### 5. 文件放置位置（Cursor 优先）
- **自定义 meta-skill**：  
  `~/.cursor/skills/spec-first-superpowers/SKILL.md`（全局）  
  或项目内 `.cursor/skills/spec-first-superpowers/SKILL.md`（项目专用）

- **Standalone Prompt / 项目规则**（强制注入）：  
  项目根目录 `.cursor/rules/00-spec-first-superpowers.mdc`  
  （alwaysApply: true，确保每个会话生效）

- **项目宪法（constitution.md）**：  
  - OpenSpec 项目（默认）：参考复制到 `.openspec/config.yaml` 的 `context` / `rules` 字段，或保存为独立 md 文件  
  - Spec-Kit 项目：`.specify/memory/constitution.md`

#### 6. 如何启动与使用
1. 打开 Cursor 项目，新建或继续聊天。
2. 输入激活命令：  
   `/super-spec` 或 `@super-spec` 或 “use super-spec”  
   → AI 强制输出启动模板，并开始完整流程：
   - 智能判断模式（默认 OpenSpec）
   - 创建/读取 `.spec-mode`
   - 进入 SDD 规范（OpenSpec 优先：/opsx:new 等）
   - planning-with-files 生成持久文件
   - UI/UX 检测 → 生成设计系统（若涉及）
   - Superpowers TDD + 审查执行

3. **典型使用示例**  
   你：我想加一个用户仪表盘，支持拖拽小部件。  
   AI：（强制模板）  
   “我已激活 Spec-First + Superpowers 终极版（默认偏向 OpenSpec）。  
   当前模式：OpenSpec（记录在 .spec-mode）。  
   完整流程即将启动：  
   1. SDD 规范先行  
   2. planning-with-files 持久规划  
   3. UI/UX 设计系统（已检测到仪表盘）  
   4. Superpowers TDD 执行  
   请确认需求细节，我立即开始 /opsx:new user-dashboard-with-drag-widgets”

4. **强制文件依赖**  
   - task_plan.md / findings.md / progress.md（根目录或 .cursor/ 下）  
   - OpenSpec 项目：.openspec/changes/xxx/（proposal.md、spec.md、design.md、tasks.md）  
   - UI 设计系统：可保存为 design-system.md

#### 7. 常见问题 & 维护建议
- 未触发？重启 Cursor，确保 rules/skills 路径正确。
- 模式错误？`/super-spec reset` 重置 .spec-mode。
- 缺少命令？重新运行 install-all.sh。
- UI 未自动触发？在需求中明确写 “dashboard UI” “拖拽界面” 等关键词。
- 后期维护：
  - 更新工具：定期 git pull 对应仓库，或重新运行 install 命令。
  - 版本记录：建议在项目 README 添加“AI 约束系统版本：v1.0 - 2026-02-24”。
  - 文档维护：可将本说明 + 所有 GitHub 链接保存为项目 docs/ai-constraint-workflow.md。
